What the repo says

Grounding Context

Codebase Context. Interactive calls with an omitted selection currently go straight to provider discovery. The alias store can already return the full validated alias map, but the application only injects resolve/save operations. Provider and model options preserve runtime order, and the numbered menu has two visual owners: diagnostic rendering plus a separate numeric question.

Generation writes the response exactly once and byte-for-byte to stdout. Interactive prompts use stderr, but the save prompt begins immediately after generation; when the response lacks a trailing newline, both streams share the same terminal row. The existing finish flow asks for confirmation, then asks for a name, then optionally asks about overwrite.

Current tests and documentation protect the valuable parts of the contract: explicit aliases bypass discovery, non-interactive invocations stay deterministic, pre-generation cancellation exits 130, post-success skipping exits 0, overwrites require consent, and even an absent trailing newline is part of exact stdout.

External prior art is aligned: Clack autocomplete searches structured option text, hints, and values; AWS auto-prompt and fzf normalize type-to-filter with arrows and Enter; mature LLM CLIs treat aliases as first-class persistent shortcuts. For app-owned ANSI styling around the prompt, picocolors provides a tiny, dependency-free, NO_COLOR-friendly ESM API.

Non-negotiable constraint

The visible newline belongs to the interactive UI channel. Appending it to stdout would fix the screenshot while breaking a stronger shell contract.

1

The named execution loop

Axis · Cross-axisConf · 92%Cx · Medium

Treat aliases as a reinforcing product loop rather than a detached flag. An omitted-alias interactive run begins with saved names; fresh discovery remains one explicit escape hatch; after a fresh run, one optional field can name that choice. The selected result carries whether it was already named, so existing aliases finish cleanly and only fresh choices invite capture.

Basis
direct: loadAliases() already exposes the complete map; current selection paths already converge on the same provider/model record; current save eligibility cannot represent an alias picked from an interactive screen. external: mature CLI profile and alias systems make saved configurations the repeat-use fast path.
Rationale
This is the only direction that changes the compounding behavior of the product: each exploratory choice can make every later run faster.
Downsides
Touches selection provenance, alias loading, prompt UI, and finish behavior together. It should be defined as one coherent experience before implementation.
2

Alias library as the default home

Axis · Startup routingConf · 96%Cx · Medium

When aliases exist, show one searchable alias picker before discovery. Each row uses the alias as its label and provider · raw model as a hint, with a final “Choose a provider and model…” option. With no aliases, skip the screen entirely and open the provider picker.

Basis
direct: The alias document is already loadable and missing storage already means an empty map, not an error. Explicit aliases already bypass discovery. external: AWS profiles, navigation saved destinations, and command palettes all privilege repeat destinations while preserving an escape hatch.
Rationale
Users stop memorizing alias spellings, and repeat runs avoid provider discovery without penalizing first-run users.
Downsides
Alias records do not store friendly model labels. Showing those would require discovery and defeat the fast path, so hints should use stored provider and raw model identity.
3

One typed autocomplete contract

Axis · Searchable selectionConf · 94%Cx · Medium

Use one structured option shape—{ value, label, hint }—and one Clack autocomplete adapter for aliases, providers, and models. Let the prompt own rendering, filtering, arrows, Enter, and cancellation; remove the separately printed numbered menu. Model rows show the friendly label and expose raw ID as hint/search text so filtering never weakens identity.

Basis
direct: The current menu is printed once through diagnostics and controlled again through numeric input; model identity is recovered by indexing the original array. external: Clack autocomplete supports structured, searchable options and cancellation.
Rationale
One interaction language fulfills the typeahead request and removes the wrong-model risk that comes from sorting labels separately from values.
Downsides
The adapter must be tested against native compiled binaries and must direct Clack UI to stderr. Cancellation needs explicit mapping to current exit behavior. Let Clack own list styling; reserve picocolors for app-owned context and success text.
4

Deterministic recognition order

Axis · Searchable selectionConf · 98%Cx · Low

Sort copied alias, provider, and model option arrays case-insensitively by visible label, with the raw value or ID as a deterministic tie-breaker. Filtering helps recall after typing; stable order helps recognition before typing. Both behaviors should be public UX promises.

Basis
direct: Provider and model arrays currently pass runtime order straight into the UI, while BYOK explicitly leaves UI-specific model sorting to host applications.
Rationale
Identical choices become predictable across runs and machines, while typed values preserve the correct runtime identity after sorting.
Downsides
Alphabetical order intentionally discards any upstream ranking. The comparator must specify case and duplicate-label tie behavior.
5

A deliberate terminal epilogue

Axis · Terminal boundaryConf · 95%Cx · Low

After writing the exact model response to stdout, open post-generation UI on stderr with a small, deliberate vertical gutter. Use the response’s trailing-newline state to choose the minimum ordinary-text separator, but frame the promise as “the Clack prompt starts on a clean line,” not as a brittle byte recipe.

Basis
direct: The app writes the full response and immediately prompts; exact stdout explicitly includes an absent trailing newline; interactive UI already belongs to stderr.
Rationale
This fixes the observed collision without contaminating pipes, redirected output, or byte-level tests.
Downsides
Carriage returns, terminal control sequences, and stream interleaving complicate a universal visual guarantee. The downstream design should test real terminal cases instead of assuming newline detection covers all output.
6

One-field save-or-exit with contextual collisions

Axis · Finish flowConf · 91%Cx · Medium

After fresh discovery only, ask exactly what to type and what will be named: “Enter an alias name for OpenAI · gpt-3.5 (Enter to exit),” with a dim example such as “e.g. fast.” Empty or cancel exits successfully; a name validates and saves. If the name already maps to the same target, report “Already saved”; if it maps elsewhere, show old and new targets before asking to overwrite.

Basis
direct: The current flow asks permission before asking for the name; skipped post-success saving already exits 0; the alias store already exposes the current record during collision handling. external: picocolors supplies green/dim ANSI formatting while honoring unsupported-color and NO_COLOR environments.
Rationale
Both common outcomes become one action—Enter to finish or type a name—without sacrificing overwrite safety. Use picocolors green for the prompt and saved confirmation, and dim styling for the example.
Downsides
Inline validation and retaining attempted input depend on Clack text-prompt behavior. Equality handling must remain race-safe inside the store’s locked save path.
7

Recovery-aware Clack picker

Axis · Searchable selectionConf · 88%Cx · Medium

Port the existing model-list recovery behavior into the searchable UI instead of losing it during the prompt-library swap. A failed provider gets a concise stage-labelled message, disappears for the current run, and the remaining sorted provider picker redraws. Ctrl-C during selection still exits 130 before generation; blank completion after output remains exit 0.

Basis
direct: The current loop filters a failed provider and continues, and manual tests require remaining choices to be offered. Existing exit codes already distinguish cancellation before generation from successful completion without saving.
Rationale
The UX migration preserves one of the current CLI’s strongest resilience behaviors while making the recovery feel like one coherent interaction.
Downsides
Clack may not preserve an in-progress search query across redraws. A fresh redraw is acceptable, but the error must remain visible without duplicating the whole menu.
